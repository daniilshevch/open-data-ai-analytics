# Система комплексного аналізу макроекономічних прогнозів України (2020-2021)

## Мета проєкту
Метою цієї роботи є:
1. Створити автоматизований конвеєр для обробки та аналізу відкритих даних за допомогою методології **Git Flow**. Проєкт охоплює етапи завантаження, перевірки якості, дослідження та візуалізації економічних показників.
2. Забезпечити 100% точність даних через валідацію та очищення.
3. Виявити приховані закономірності в прогнозах економічного розвитку України.

## Джерело даних
У проєкті використовуються дані з Єдиного державного веб-порталу відкритих даних [data.gov.ua](https://data.gov.ua/).
**Набір даних:** Прогнозні макропоказники економічного і соціального розвитку України на 2020-2021 роки (за різними сценаріями).

## Питання та гіпотези для аналізу
Аналіз зосереджений на наступних гіпотезах:
1. **Порівняння сценаріїв:** Наскільки суттєво відрізняється прогноз номінального ВВП між найбільш оптимістичним (Сценарій 2) та песимістичним (Сценарій 3) прогнозами на 2021 рік?
2. **Аналіз інфляційних очікувань:** Як змінюється індекс споживчих цін (грудень до грудня) залежно від обраного сценарію розвитку економіки?
3. **Соціальний добробут:** Порівняння прогнозів заробітної плати між сценаріями.

## Структура репозиторію
* `data/` — сирі та оброблені дані (ігноруються Git).
* `notebooks/` — Jupyter Notebooks для експериментів.
* `src/` — модулі `data_load`, `data_quality_analysis`, `data_research`.
* `reports/figures/` — згенеровані графіки та звіти.
* `infra/` — конфігураційні файли Terraform та Cloud-init.

## Інфраструктура та автоматизація розгортання (IaC)
Для забезпечення відтворюваності середовища та автоматизації розгортання обчислювальних ресурсів у проєкті використано методологію **Infrastructure as Code (IaC)** на базі **Terraform** та **Microsoft Azure**.

### Налаштування середовища через Azure Cloud Shell
Для керування інфраструктурою використовується вбудований термінал Azure:
1. **Запуск:** Натисніть іконку Cloud Shell (`>_`) на верхній панелі порталу Azure.
2. **Підготовка:** Перейдіть у папку з конфігураційними файлами (`.tf`) та переконайтеся у наявності SSH-ключа (`~/.ssh/id_rsa.pub`).

### Життєвий цикл керування інфраструктурою
Розгортання виконується за допомогою наступних команд:

* `terraform init` — ініціалізація проєкту та завантаження провайдерів.
* `terraform validate` — перевірка синтаксису конфігурації.
* `terraform apply -auto-approve` — автоматичне створення ресурсів (VM B2s, NSG, VNet) та запуск сервісів через **Cloud-init**.
* `terraform destroy -auto-approve` — повне видалення ресурсів після завершення роботи.
